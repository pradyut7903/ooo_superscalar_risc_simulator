# Experiments (toy-trace suite)

Older IPC sweeps on the custom `.trace` ISA / benchmarks under `benchmarks/`.
For the current hex + cached-memory sizing numbers, see
[results/SIZING_STUDY.md](results/SIZING_STUDY.md).

Build at `-O0` (already set in CMake). GCC 16.1.0 was miscompiling this sim at
`-O1`/`-O2`.

## What it models

Configurable-width OoO core: Tomasulo + ROB + RAT, gshare/BTB + RAS, LSQ with
store-to-load forwarding, ALU/MUL/DIV into a CDB. Structure sizes and latencies
are CLI knobs. Stats via `--stats` / `--csv`.

Toy traces also have `ADDI` and a `.mem ADDR VALUE` preload so the kernels can
set up constants, counters, and arrays.

## Benchmarks

Generated by `tools/gen_benchmarks.py`. Each has a `.expect` file checked by
`tools/check_benchmarks.py`.


| Benchmark                 | Kernel               | Stresses                |
| ------------------------- | -------------------- | ----------------------- |
| `vecadd`                  | `C[i]=A[i]+B[i]`     | mem bandwidth, LSQ, ILP |
| `saxpy`                   | `y[i]=a*x[i]+y[i]`   | MUL + load/store        |
| `dotprod`                 | `sum += A[i]*B[i]`   | serial reduction        |
| `dotprod_unroll`          | 4 accumulators       | more ILP than serial    |
| `matmul`                  | 12×12 multiply       | nested loops, reduction |
| `ptrchase`                | `p = MEM[p]`         | load-use chain          |
| `ilp_serial`              | one dependent chain  | IPC floor               |
| `ilp_parallel`            | 4 independent chains | IPC ceiling             |
| `mul_heavy` / `div_heavy` | independent MUL/DIV  | FU contention           |


```
python tools/gen_benchmarks.py
python tools/check_benchmarks.py
```

## Metrics

`--csv` prints cycles, commits, IPC, commit mix, branches/mispredicts/flushes,
and stall counters (`stall_rob`, `stall_rs`, `stall_lsq`, `stall_front`). The
stalls are usually more useful than IPC alone.

## Sweeps

```
python tools/run_sweep.py --study all
python tools/analyze.py --study all
```

Per-study CSVs go to `results/<study>.csv`. Individual studies: `--study rob`,
etc.

Rough groups:

- Structure size: `--rob`, `--rs`, `--lsq`, `--ifq`
- Width / FUs: `--width`, `width_fixedfu`, `--alu` / `--mul` / `--div`
- Memory: `--mem-lat`, `--fwd`

## Findings (from the CSVs in `results/`)

**ROB.** On `vecadd`, IPC rises ~1.30 → 2.77 as ROB goes 8 → 24, then flats;
`stall_rob` hits 0 at ROB=24. `ptrchase` barely cares (bottleneck is the
load-use chain).

**Width.** `ilp_parallel` scales with width; `ilp_serial` tops out around ~3;
`matmul` stays ~1–2 (MUL + reduction). Width only helps if the code has ILP.

**FUs.** `mul_heavy` / `div_heavy` scale with unit count. Kernels that aren't
FU-bound saturate early (often at 2 MULs).

**Mem latency.** `ptrchase` collapses as latency grows (serial chain). `vecadd`
holds up better because independent loads overlap in the LSQ. LSQ depth matters
for `vecadd` (~12 entries to peak); `ptrchase` does not.

## Bugs the benches caught

Short directed traces missed these; the looped benches hit them. All fixed:

1. Mispredict recovery ran at resolve instead of commit, killing older work and
  deadlocking loops. Deferred to ROB head; squash only younger.
2. Operands were re-read from the RAT every cycle, so an op could latch onto a
  wrong (younger) tag. Capture at issue; wake from CDB only.
3. Mispredict checked direction but not target, and the BTB wrote fall-through
  PCs on not-taken branches — wrong-path commits at width ≥ 3. Check target;
   only update BTB on taken.
